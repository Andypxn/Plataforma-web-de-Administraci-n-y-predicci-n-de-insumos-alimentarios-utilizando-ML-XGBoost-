Ingeniería de Optimización: Ensambles y Descenso del Gradiente

Módulo: Core Predictivo XGBoost

Objetivo: Análisis Matemático de la Reducción del Error Residual

1. La Naturaleza del Ensamble Aditivo

XGBoost (eXtreme Gradient Boosting) basa su poder en la combinación de múltiples modelos "débiles" (árboles de decisión pequeños) para formar un modelo "fuerte" con alta capacidad de generalización sobre el 80% de los datos de entrenamiento.

¿Qué sucede en el entrenamiento?

No se construye un árbol complejo de una sola vez. Se agregan secuencialmente. Cada nuevo árbol se encarga de explicar lo que los árboles anteriores no pudieron capturar.

Defensa Técnica: "No estamos promediando árboles (como en Random Forest), estamos sumando correcciones de forma aditiva (Boosting)".

2. El Ciclo de Aprendizaje de Pesos

El proceso utiliza la técnica de Gradient Descent para minimizar una Función de Pérdida (Loss Function).

L(y, ŷ) = Σ (yi - ŷi)2 + Ω(fk)
(Error Cuadrático + Término de Regularización)
Paso a Paso del Algoritmo:
  1. Predicción Inicial: Se asigna un valor base (la media de la demanda en el set del 80%).
  2. Cálculo de Residuos: Se mide la diferencia entre la demanda real y la predicha (Error).
  3. Ajuste de Pesos: El nuevo árbol asigna "pesos" a las hojas para predecir estos residuos específicamente.
  4. Descenso del Gradiente: Se calcula la dirección (gradiente) en la que el error disminuye más rápido y se actualiza el modelo en esa dirección.

3. Descenso del Gradiente: Minimizando la Función de Error

Para cada nuevo árbol, XGBoost calcula el gradiente (derivada de la función de pérdida) para saber qué tan lejos está de la realidad.

graph TD Start[Predicción Árbol N] --> CalcErr[Calcular Gradiente del Error] CalcErr --> Update[Ajustar Pesos del Árbol N+1] Update --> LearningRate[Aplicar Tasa de Aprendizaje - Eta] LearningRate --> Combine[Sumar al Ensamble Final]
Concepto Maestro: El eta (learning rate) actúa como un amortiguador. No permitimos que un solo árbol cambie todo el modelo; cada uno contribuye un pequeño porcentaje, lo que garantiza estabilidad y evita el sobreajuste.

4. Verificación de Precisión (80% vs 20%)

Una vez que el ensamble de árboles ha terminado su ciclo de optimización sobre el set de entrenamiento, procedemos a la fase de validación estricta.

Conjunto Función Resultado Esperado
Entrenamiento (80%) Ajuste de gradiente y pesos Minimización del error residual cercano a cero.
Test/Validación (20%) Evaluación de generalización Comparación de ŷ (predicho) vs y (real) mediante MAE/RMSE.
Métrica de Defensa:

Si el error en el 80% es muy bajo pero en el 20% es muy alto, el descenso del gradiente fue "demasiado agresivo" (Overfitting). El éxito de nuestro sistema radica en que ambos errores sean similares y bajos.
