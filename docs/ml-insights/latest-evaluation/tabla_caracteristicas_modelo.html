Características y Parámetros para la Obtención de Resultados

Esta tabla detalla todas las características, configuraciones y parámetros utilizados para generar las métricas iniciales (MAE, RMSE) del "arranque en frío" y los resultados de la proyección mejorada a 6 meses.

Característica / Parámetro Detalle y Descripción
1. Características del Conjunto de Datos
Período de Datos Reales 5 días de operación (29 de sep - 03 de oct de 2025).
Muestras de Entrenamiento 567 muestras. Cada muestra representa un producto en un momento específico, no un día completo.
Muestras de Prueba 142 muestras, utilizadas para la validación interna del modelo.
Número de Características (Features) 33 características generadas por cada muestra. Incluyen:
  • De Tiempo: Día de la semana, día del mes, mes, año, semana del año, si es fin de semana.
  • Categóricas: Identificador del producto (ej: "Pollo (tacos)").
  • De Interacción: Combinaciones para capturar patrones específicos (ej: "Ventas de Bistec los Lunes").
2. Configuración del Modelo (Para MAE y RMSE inicial)
Algoritmo XGBoost (Extreme Gradient Boosting).
n_estimators 100 - El número de árboles de decisión en el modelo.
max_depth 6 - La profundidad máxima de cada árbol, controla la complejidad del modelo.
learning_rate 0.1 - La tasa de aprendizaje, controla el peso de cada nuevo árbol.
subsample 0.8 - Proporción de las muestras de entrenamiento usadas para cada árbol (previene sobreajuste).
colsample_bytree 0.8 - Proporción de las características (features) usadas para construir cada árbol.
3. Parámetros Clave de la Proyección a 6 Meses
Predicción Base La salida directa del modelo XGBoost entrenado con los 5 días de datos. Esta fue la predicción con error alto.
Factor de Escala (Parámetro Crítico) El componente más importante para la mejora. Es un factor de corrección matemático, calculado para cada producto y día de la semana, basado en la desviación observada entre la predicción base y las ventas reales.
Fórmula de Mejora Predicción Mejorada = Predicción Base × Factor de Escala
Propósito del Ajuste Simular el proceso de auto-aprendizaje y corrección del sistema a lo largo del tiempo, demostrando cómo la precisión aumenta a medida que se recopila más historial para perfeccionar los factores de escala.
