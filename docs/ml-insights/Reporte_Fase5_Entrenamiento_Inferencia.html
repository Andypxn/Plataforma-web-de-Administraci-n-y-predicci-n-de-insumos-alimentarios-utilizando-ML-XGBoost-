Fase 5: Entrenamiento y Estrategia de Inferencia

Objetivo: Optimización del Modelo XGBoost y Definición del Horizonte de Predicción

Tesis: Sistema de Abastecimiento Inteligente para Restaurantes

1. Particionamiento Estratégico (80/20)

Para garantizar que el modelo sea capaz de generalizar (predecir el futuro) y no solo memorizar el pasado (overfitting), dividimos el dataset en dos conjuntos críticos.

graph LR A[Dataset Total: 6 Meses] --> B[Entrenamiento - 80%] A --> C[Pruebas/Test - 20%] B --> D[El Modelo Aprende Patrones] C --> E[El Modelo es Evaluado]
Configuración Técnica:
  • Set de Entrenamiento (80%): Aproximadamente 144 días. Se utiliza para ajustar los pesos de los árboles de decisión.
  • Set de Prueba (20%): Aproximadamente 36 días. Son datos "invisibles" para el modelo; se usan para simular la efectividad en el mundo real.
Justificación: En series temporales, este split no es aleatorio (shuffle), sino cronológico. Entrenamos con los primeros meses para predecir el último mes, validando que el modelo entiende la evolución del negocio.

2. Dinámica del Entrenamiento (Gradient Boosting)

XGBoost no construye un modelo perfecto de golpe, sino que lo perfecciona iterativamente mediante una técnica llamada Boosting.

ModeloFinal = Árbol1 + η·Árbol2 + η·Árbol3 + ...
¿Cómo aprende el modelo?
  1. El primer árbol hace una predicción base (promedio).
  2. Se calculan los Residuales (el error cometido).
  3. El segundo árbol se entrena específicamente para corregir los errores del primero.
  4. Este ciclo se repite hasta minimizar el error total (MAE/RMSE).
Impacto: Esta corrección sucesiva permite que el sistema detecte patrones sutiles que otros algoritmos (como las regresiones lineales) ignorarían.

3. El Horizonte de Predicción (Look-ahead Window)

Un modelo de abastecimiento debe ser útil para la logística. Predecir "mañana" no es suficiente si el proveedor entrega cada semana.

Configuración del Horizonte:
  • Horizonte de Predicción: 7 días (Plan Semanal).
  • Frecuencia: El modelo se ejecuta cada noche para actualizar las proyecciones de los próximos 7 días.
Justificación de Negocio: El periodo de 7 días permite al restaurante realizar un pedido consolidado, optimizando costos de transporte y asegurando stock suficiente para el ciclo completo de estacionalidad (Lunes-Domingo).

4. Proceso de Inferencia (Predicción Real)

Para generar la predicción de los próximos 7 días, el modelo utiliza como entrada los datos más recientes del sistema POS.

sequenceDiagram POS->>Feature Engineering: Envía ventas de hoy (T) Feature Engineering->>XGBoost: Crea Vector con Lags (T-1, T-7) XGBoost->>Dashboard: Entrega Demanda Estimada (T+1 a T+7) Dashboard->>Usuario: Sugerencia de Compra de Insumos
Métrica de Éxito: Durante la validación con el 20% de los datos, comparamos la predicción con lo que realmente se vendió. Si el error (MAE) es menor al 10%, el modelo se considera apto para la operación.
